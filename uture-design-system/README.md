# UTURE Design System

UTURE(유쳐)는 **AI · 기술 입문 교육 콘텐츠 에이전시**(UTURE AX Agency)이다.
법인명은 **uture (아이솔)**. 본 디자인 시스템은 UTURE 강의용 PPT, 워크북, 학습 자료의 시각 언어를 정의한다.

> "잘 정리된 교재처럼 보일 것."
> 마케팅 데크 · 영상 썸네일 · 블로그의 톤이 묻으면 안 된다.

---

## 1. Brand DNA

UTURE는 한국 학습자를 대상으로 **AI/기술 입문 강의**를 만든다.
대상은 비전공 직장인 · 개발 외 직군 · 학생. 따라서 자료는 화려한 마케팅 톤이 아니라
"교과서 + 에디토리얼 매거진 + 노트 정리"의 중간 지점을 노린다.

**다섯 가지 원칙**

1. **여백은 도구지 목표가 아니다** — 여백은 위계를 만들기 위한 수단. 빈 슬라이드가 목적은 아니다. 본문 영역(상단 96 · 하단 96 · 좌우 120 안쪽)의 **40–70%는 콘텐츠로 채운다.** 채울 게 없으면 슬라이드를 합치거나 시각 요소를 더한다.
2. **한 개념, 필요하면 여러 장** — 한 슬라이드 한 메시지. 단, 복잡한 개념은 여러 슬라이드에 펼친다.
3. **개념마다 시각 장치 1개** — 정의 슬라이드라도 비유 그림·도형·표·인포그래픽 중 **하나는 반드시 동반한다.** 텍스트만으로 끝나는 본문 슬라이드는 Statement·Closing 외에는 없다. (§ 5.X Visual Playbook 참고)
4. **주황색은 사건이다** — 슬라이드당 1–2회만.
5. **시각 요소는 이해를 가속할 때 우선** — 도형은 장식이 아니라 설명 가속기다. 텍스트보다 빠르게 구조를 보여줄 수 있을 때 쓴다. 단, 의미 없는 아이콘·클립아트·플레이스홀더 박스는 텍스트보다 나쁘다.

**절대 하지 않는 것**

- 그라데이션 / 그림자 / 광택 / 풀블리드 사진
- 의미 없는 아이콘 · 일러스트 · 이모지
- 영상 썸네일 같은 큰 제목+부제+사진 조합
- 한 슬라이드 3가지 이상 강조색

## 4. CONTENT FUNDAMENTALS

UTURE 카피는 "강의 노트"의 톤이다. 친근한 영상 멘트·블로그의 어투를 슬라이드에서 걷어낸다.

### 톤 & 보이스

- **언어:** 한국어 본문 + 영문 약어/기술 용어. 영문 처음 등장 시 풀어 쓴 후 약어 정착: `RAG (Retrieval-Augmented Generation)`.
- **시점:** 1인칭("저는", "우리는") 금지. 2인칭("여러분", "당신") 금지. **무인칭 기술 어투**.
- **종결:** 명사형 종결("~의 핵심") 또는 짧은 동사형("~한다", "~된다"). 친근형("~해보세요") 금지.
- **호칭:** 학습자는 "수강생" 또는 호칭 생략.
- **마침표:** 슬라이드 제목에는 찍지 않는다. 본문 문장에는 찍는다.
- **물음표:** Statement 슬라이드에서만 허용 ("RAG는 정말 죽었는가?").
- **느낌표:** 사용 금지.

### 강조

- **굵게 (600/700):** 단락당 1–3개의 핵심 개념어.
- **주황색 (`--ut-accent`):** 진짜 키워드, 슬라이드당 **1개 원칙**.
- **밑줄:** 링크 외 금지.
- **이탤릭:** 한글 금지. 영문 논문/인용에만.
- **트래킹 대문자:** `eyebrow` (챕터 라벨)에만. 본문 대문자 변환 금지.

### 금지 표현

- "꿀팁", "필수", "반드시 알아야 할", "OOO의 모든 것"
- "혁신적인", "차세대", "AI 시대" 류 마케팅 형용사
- "구독", "좋아요", "댓글", "공유" — 플랫폼 멘트
- "제가 ~ 해봤더니" — 개인 경험담 / 자랑

### 카피 예시 (Good vs Bad)

| ✅ Good (슬라이드용) | ❌ Bad (영상·블로그용) |
| --- | --- |
| 인베딩: 텍스트를 숫자 위치로 바꾸기 | 인베딩이 뭐길래 다들 이렇게 난리일까요? |
| RAG의 한계: 검색 품질이 답변 품질을 지배한다 | RAG의 진짜 단점, 이것만 알면 끝! |
| 03 ─ RAG의 기본 구조 | 오늘 배워볼 RAG 완전 정복 |
| 좋은 청크 = 의미 단위로 끊긴 한 덩어리 | 청크는 이렇게 나누시면 됩니다 ✨ |

### 챕터 라벨 (eyebrow)

- 형식: `00 ─ 짧은 명사구`. 8–14자 권장.
- 일관성: 한 강의 안에서 모든 챕터명은 같은 톤·길이를 유지.

## 5. VISUAL FOUNDATIONS

### 색 (Color)

- **캔버스는 항상 `#ffffff`.** 색 배경 슬라이드 금지. 예외: Section Divider만 `--ut-ink-strong (#0d0707)` 검정 배경 허용.
- **단 하나의 강조색:** `--ut-accent #ff4f00` (warm orange). 보색·삼색 배합 금지.
- **회색 위계 4단계:** `--ut-ink` 본문 → `--ut-text-soft` 보조 → `--ut-text-muted` 캡션 → `--ut-line` 구획선. 5번째 회색 만들지 않는다.
- **세컨더리 의미색:** good `#1f6b4a` / bad `#8a3a2a` / info `#2a4e8a` — Before/After 비교에만, 그 슬라이드의 주황은 줄인다.

### 타입 (Type)

- **표지·섹션·본문 전체:** Pretendard 단일 패밀리 (한국어/영문/숫자 비례 최적). 9 weights를 위계로 활용.
- **코드/벡터값:** JetBrains Mono (코드, 모델명 `gpt-4o-mini`, 임베딩 배열 등).
- **스케일:** display 112 / title-1 72 / title-2 56 / title-3 40 / lead 36 / body 30 / body-sm 26 / caption 22 / eyebrow 20 / code 26. 교육 가독성을 위해 본문·캐프션 티어는 +8px.
- **정렬:** 좌측 정렬이 기본. 표지·섹션 디바이더 외에는 가운데 정렬 금지.
- **본문 최대 폭:** 캔버스의 70% (1920 기준 약 1340px).

### 간격 (Spacing · 8pt)

`8 / 16 / 24 / 32 / 48 / 64 / 96 / 120`. 슬라이드 세이프 마진은 좌우 120 · 상하 96.

### 모서리 / 보더 / 그림자

- **Radius:** 카드 12 · 칩 999 · 아이콘 박스 8.
- **보더:** 1px `--ut-line` 기본. 강조 1.5px `--ut-accent` 또는 `--ut-ink`.
- **그림자: 사용하지 않는다.** 깊이는 보더와 배경 톤 차이로 만든다.

### 배경 · 이미지

- **풀블리드 사진 / 스톡 이미지 / 그라데이션 / 텍스처: 모두 금지.**
- 캔버스는 흰색. 표면 변주는 `--ut-surface-1 (#fafafa)`와 `--ut-surface-2 (#f5f5f3)` 두 단계만.
- 사진을 슬라이드에 사용해야 할 경우: 작게, 보더와 함께, 본문 1/3 이내.

### 로고 사용 (Logo Usage)

UTURE는 **두 개의 마크**를 운용한다. 둘 다 단색·고정 비율이며 회전·기울기·테두리 추가·그라데이션 모두 금지.

| 자산 | 경로 (1순위 / 대체) | 형태 | 용도 |
| --- | --- | --- | --- |
| **브랜드 마크** | `assets/uture_logo.png` · `assets/uture-logo.svg` | 비대칭 4점 스파클 | 표지·섹션·클로징의 코너 앵커 |
| **워드마크** | `assets/uture_text_logo.png` · `assets/uture-text-logo.svg` | 소문자 "uture" | 표지·섹션·클로징의 브랜드 서명 |

> 슬라이드 코드는 **PNG**를 1순위로 참조한다 (PPT·인쇄·미리보기 어디서나 깨지지 않음). SVG는 동일 아트워크의 벡터 사본으로, 색상 조작·확대 출력이 필요할 때만 교체 사용. PNG 사용 시 `object-fit: contain`을 함께 지정해 비율 보호.

**배치 규칙**

워드마크(`uture_text_logo`)는 **데크 전체에서 단 한 번, 표지에서만** 등장시킨다. 다른 모든 슬라이드는 그래픽 마크 단독 또는 무(無) 로고로 처리한다. 브랜드 마크와 워드마크가 한 화면에 둘 다 보이는 슬라이드가 표지 외에 생기면 잘못된 사용이다.

- **표지 (Cover):** 마크 + 워드마크를 **마스트헤드 페어**로 우상단에 세로 스택. 마크 104×104 / 워드마크 높이 36 / 두 요소 사이 간격 20px. 좌측 본문(타이틀)과 충돌하지 않도록 우측 120px 세이프 안쪽으로. **워드마크는 이 위치 단 한 곳에서만 사용한다.**
- **섹션 디바이더 (Section Divider, dark):** 우상단에 마크 64×64 **흰색** (`filter: invert(1)`)만. 본문 블록 하단에는 **4×96 주황 라인**으로 마무리. 워드마크는 두지 않는다.
- **클로징 (Closing):** 우상단에 마크 72×72 단독. 표지보다 한 단계 작게. 워드마크는 두지 않는다 — 클로징은 마크와 카피라이트 푸터의 `uture(유쳐) by 아이솔` 텍스트로 충분하다.
- **본문 슬라이드 (Concept / Comparison / Flow / Step / Statement 등):** 로고를 두지 않는다. 본문에서는 eyebrow가 브랜드 위치를 대신한다. 페이지 번호 외 추가 브랜딩 금지.
- **워크북·핸드아웃 (A4 인쇄):** 표지에 마스트헤드 페어, 마지막 페이지에 워드마크 단독(서명 용도). 본문 페이지에는 워드마크를 반복 노출하지 않는다 — 헤더 라인이 필요하면 그래픽 마크 16px 또는 텍스트로만.

**최소 크기 · 클리어 스페이스**

- 마크 최소 24×24px (디지털) / 8mm (인쇄). 그 이하로 줄이지 않는다.
- 워드마크 최소 높이 16px (디지털) / 6mm (인쇄).
- 두 마크 모두 자기 높이의 **0.5배** 이상의 빈 공간을 사방에 확보한다.

**색**

- 기본: `--ut-ink-strong` (#0d0707) 또는 순흑.
- 다크 캔버스 위: 순백. SVG는 `filter: invert(1)`로, PNG는 별도 화이트 파일이 필요하면 추후 추가.
- **주황(`--ut-accent`)으로 칠한 로고는 금지.** 강조색은 슬라이드의 다른 한 점이 차지한다 (§ 1 원칙 4).
- 회색조 그러데이션, 음각/양각, 광택, 그림자 모두 금지.

**절대 하지 않는 것**

- 마크를 워드마크 글자 사이에 끼우는 합성 (예: 'u-MARK-ture')
- 마크 회전 (45° 비스듬히 두기 등) — 비대칭 스파클의 의도된 기울기를 깬다
- 마크·워드마크를 텍스트 인라인으로 사용 (제목 안에 박는 등)
- 색 배경 위 로고를 박스·라운드 컨테이너로 감싸기

### 애니메이션 / 트랜지션

- **슬라이드 전환은 단순 cut 또는 짧은 dissolve(120ms).** Bounce, slide-in, parallax 금지.
- 화면 내 강조 요소는 등장 시 fade-in 200ms · ease-out 정도. 그 이상은 강의 흐름을 방해.

### 인터랙션 (PDF가 아닌 인터랙티브 자료용)

- **Hover:** 1.5px 보더가 `--ut-line-strong`로 짙어지거나, 텍스트가 `--ut-ink → --ut-ink-strong`. 색 반전 금지.
- **Press:** scale 0.98 · 100ms ease-out. 그림자 추가 금지.
- **Focus:** 2px `--ut-accent` outline, offset 2px.

### 투명 / 블러

- **사용하지 않는다.** 슬라이드는 평면이다. 백드롭 블러, 반투명 레이어, 글래스모피즘 모두 금지.

### 카드 (Card)

`background: #fff; border: 1px solid var(--ut-line); border-radius: 12px; padding: 32px;` — 그림자 없음.

### 다이어그램 / 도형

- 도형 채움: 흰 또는 `--ut-surface-1`. 보더 1.5px `--ut-line`.
- **강조 노드 1개만** `--ut-accent` 보더 + 주황 텍스트.
- 선/화살표 1.5px `--ut-ink`. 점선 필요 시 `stroke-dasharray: 4 4`.
- 3D 도형, 입체 화살표, 손글씨 풍 선 모두 금지.

### 레이아웃 (Slide canvas)

- **1920 × 1080 (16:9).** 좌우 120 · 상하 96 세이프.
- **12 컬럼, 거터 32.** 본문은 6/12 또는 8/12 그리드 안에서.
- **본문 슬라이드는 2-zone 레이아웃을 기본으로 한다.** 좌측은 텍스트(제목 + 짧은 본문), 우측은 시각 요소(도식·표·비유 그림). 50/50 또는 6/12 + 6/12. 한쪽이 비면 위계가 약해지므로 빈 zone은 만들지 않는다.
- **모든 본문 슬라이드의 4개 앵커:**
  - 좌상단 eyebrow (mono 챕터 번호 + 하이라인 + Pretendard 500 챕터명)
  - 슬라이드 제목 (좌측 정렬, `title-2`)
  - 본문 영역 (`lead → body`) **+ 동반 시각 장치 1개**
  - **하단 푸터**
    - **표지 / 섹션 / Statement / Closing** — 좌측 `Copyright © 2026 uture(유쳐) by 아이솔. All rights reserved.` + 우측 `14 / 36` 전체 표기
    - **본문 페이지 (Concept, Comparison, Flow, Step, Definition, Matrix 등)** — 우하단 작은 `14 / 36` 번호만, 슬라이드 구말에 붙여서. 본문과 겹치지 않도록 콘텐츠는 그 위로.

### 비유 & 시각 장치 선택법 (Visual Playbook)

본문 슬라이드는 **콘텐츠 유형 → 시각 장치**의 매핑을 따른다. 텍스트만 떠 있는 슬라이드는 만들지 않는다. 아래 표에서 가장 가까운 유형을 골라 그 장치를 동반시킨다.

| 콘텐츠 유형 | 1순위 시각 장치 | 형태 | 적용 예 |
| --- | --- | --- | --- |
| **정의 (Definition)** | 비유 그림 + 키워드 핀 | 단순 선화 + 주황 1점 | "임베딩 = 의미 좌표" → 2D 산점도 |
| **구성 요소 (Anatomy)** | 분해도 (Exploded view) | 라벨 붙은 부분도 | "프롬프트의 4 파트" → 박스 4개 + 화살표 라벨 |
| **순서 (Sequence/Flow)** | 화살표 플로우 | 좌→우 5단계 이하 박스 + 화살표 | "RAG 파이프라인" |
| **비교 (Comparison)** | 카드 그리드 2–3열 | 동일 폭 카드, 강조 1개만 주황 보더 | "파인튜닝 vs RAG vs 프롬프트" |
| **분류 (Taxonomy)** | 2×2 매트릭스 또는 트리 | 축 라벨 + 사분면 | "검색 방식: 의미 ↔ 키워드 / 정확 ↔ 빠름" |
| **수치/규모** | 미니 차트 또는 비율 막대 | 흑백 막대, 강조 1개만 주황 | "토큰 한도 사용률 73%" |
| **수치 표** | 표 (table) | 1px 보더, 헤더만 굵게 | "모델별 컨텍스트 길이" |
| **인과/조건** | 결정 트리 또는 if/then 표 | 좌측 조건 → 우측 결과 | "검색 실패 시 fallback" |
| **단계별 절차 (Step list)** | 번호 카드 4–6개 | 큰 주황 숫자 + 하이라이트 라인 | "좋은 청크 만들기 4단계" |
| **양극 스펙트럼** | 1D 슬라이더 | 좌우 라벨 + 점 위치 | "탐색 ↔ 활용 (exploration-exploitation)" |
| **인포그래픽** | 아이콘 격자 (Lucide 라인) | 3–6개 셀, 라벨 + 1줄 설명 | "벡터 DB의 5가지 사용 사례" |
| **인용/Statement** | 큰 따옴표 + 출처 | 좌측 6px 주황 보더 | 본문 안에 인용일 때만 |

**선택 규칙**

- **콘텐츠가 텍스트 3줄 이상이라면 시각 장치는 필수**다. 텍스트만으로 5줄 이상 늘어지는 슬라이드는 분할하거나 표/도식으로 압축한다.
- **시각 장치는 슬라이드당 1개**가 원칙. 두 가지를 함께 쓸 때는 한쪽을 보조적으로 작게.
- **장치를 고르고 나서 글을 줄인다.** 그림이 설명하는 부분은 본문에서 빼야 중복이 사라진다.
- **장식만을 위한 도형 금지.** 의미 없는 박스, 빈 카드, 추상 패턴은 여백보다 나쁘다.
- **주황은 시각 장치 안에서도 단 한 점.** 핀, 강조 노드, 강조 카드 보더 중 하나에만.

**비유의 톤**

UTURE의 비유는 "교과서적"이다. 일상 사물(책·서랍·지도·저울 등)을 단순 선화로 그리고, 강조 부분 한 점에만 주황을 얹는다. 캐릭터·말풍선·드로잉체·손글씨 풍은 금지.

## 6. ICONOGRAPHY

UTURE의 원칙은 **"아이콘은 없어도 된다"**이다.
텍스트와 다이어그램으로 설명할 수 있다면 아이콘은 빼는 것이 우선이다.

### 허용되는 스타일

- **단색 라인 아이콘.** Phosphor Light / Lucide / Tabler Icons 등 라인 패밀리.
- **굵기 1.5px.**
- **컬러:** `--ut-ink`이 기본. 한 슬라이드에 1개만 `--ut-accent`로 강조.
- **사이즈:** 24 / 32 / 40 px (슬라이드 본문 기준).

### 현재 시스템에서의 선택

본 디자인 시스템은 별도의 아이콘 폰트를 번들하지 않는다. 슬라이드 자료에서 아이콘이 정말 필요할 때는 **Lucide** (CDN)를 권장한다:

```html
<script src="https://unpkg.com/lucide@latest"></script>
<i data-lucide="search"></i>
<script>lucide.createIcons();</script>
```

> ⚠️ Lucide는 UTURE 사양서에 명시된 라인 아이콘 정신에 부합하는 가장 가까운 오픈 라이브러리 선택이다(stroke-width 2 기본을 1.5로 낮춰 사용). 만약 자체 아이콘 세트가 있다면 `assets/icons/`에 SVG로 추가하고 본 README의 권장을 갱신해 주세요.

### 메타포 그림

오픈북 시험 · 마트 진열 같은 **비유 그림**은 단순 선화 + 단일 강조색으로 표현한다. 디테일을 채워 그리지 않는다.
- 오픈북: 책 윤곽선 + 작은 돋보기. 돋보기 손잡이만 주황.
- 마트 진열: 가로 라인 3개(선반) + 사각형 4개. 정답 위치 1개만 주황.

### 절대 사용 금지

- 컬러풀 / 글래스 / 듀얼톤 아이콘 세트
- 비즈니스 클립아트, 캐릭터 일러스트, 3D 일러스트
- 이모지 (수업 본문/시각 자료 모두)
- 유니코드 픽토그램 (☑, ★, ➤ 등 모두 금지 — 위계 강조는 타이포로)
