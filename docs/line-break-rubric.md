# Line-break Rubric — UTURE 교안 한국어 줄바꿈 검증 기준

이 문서는 산출물(HTML 슬라이드)의 **한국어 줄바꿈이 어절을 보존하는지** 판정하기 위한 루브릭이다.

- 적용 대상: `./results/` 하위 모든 HTML 산출물
- 사용 주체: `line-break-verifier` 서브에이전트 + 메인 에이전트의 자체 점검
- 호출 순서: `visual-verifier` → `line-break-verifier` (시각 검증 통과 후 줄바꿈 검증)

---

## 0. 검증 대상 — 무엇이 "잘못된 줄바꿈"인가

한국어는 띄어쓰기 단위인 **어절(eojeol)**이 의미 단위다. 자동 줄바꿈은 어절과 어절 **사이**에서 일어나야 하며, 어절 **안**에서 끊기면 안 된다.

| 구분 | 예시 | 평가 |
| --- | --- | --- |
| ✅ 자연스러운 줄바꿈 | `점수가 \n 달라진다.` | 어절 사이에서 끊김 |
| ❌ 어절 중간 잘림 | `점수가 달라진 \n 다.` | "달라진다" 어절이 잘림 |
| ❌ 영문 단어 잘림 | `Transfor \n mer` | 단어 자체가 잘림 |
| ⚠️ 한 글자 미아 (orphan) | `다음 회차에는 다 \n .` | 마침표만 줄 단독 — 가독성 저해 |

---

## 1. 하드 룰 (위반 시 즉시 Fail)

- [ ] `colors_and_type.css` (또는 동등 스타일시트)에 `word-break: keep-all` 이 `html`/`body` 또는 본문 컨테이너에 적용되어 있다
- [ ] `overflow-wrap` 폴백이 함께 적용되어 매우 긴 단어(URL·식별자)도 캔버스를 벗어나지 않는다
- [ ] `line-break: strict` (또는 동등) — 한국어 구두점이 라인 첫머리로 밀려나지 않도록

> 위 세 가지가 모두 충족되지 않더라도, 실제 렌더에서 어절 잘림이 0건이면 PASS로 본다. 다만 시스템 단의 안전망이므로 검증자는 누락을 `Note` 로 표기한다.

---

## 2. 정량 측정 항목 (각 항목 Pass/Fail)

| # | 항목 | 측정 방법 | 임계값 |
|---|------|-----------|--------|
| 1 | **어절 잘림 슬라이드 수** | 캡처에서 한 어절이 라인 끝/시작에 걸친 슬라이드 수 | **0 장** |
| 2 | **영문/숫자 단어 잘림** | 영문 단어, 모델명, 식별자 가운데가 잘린 슬라이드 수 | **0 장** |
| 3 | **문장 끝 한 글자 미아** | "다.", "요.", ")." 등 1–2자만 단독 줄에 남은 슬라이드 수 | **≤ 1 장** (의도된 강조 라인 예외) |
| 4 | **`<br/>` 강제 줄바꿈의 자연스러움** | 명시적 `<br/>` 가 어절 가운데를 끊지 않음 | **위반 0건** |
| 5 | **구두점 라인 첫머리** | "." "," ")" "다." 가 다음 줄 첫머리로 단독 시작 | **0 장** |

> **합산 기준**: 5개 항목 중 **5개 모두 Pass** = 전체 Pass. 1개라도 Fail이면 Fail.

---

## 3. 정성 평가 항목 (각 항목 1줄 코멘트)

- **의미 단위 일치**: 줄바꿈 위치가 의미 휴지(주어/서술어/수식어 경계)와 맞는가
- **컨테이너 폭**: 본문 컨테이너 `width`/`max-width` 가 어절 보존을 깨뜨릴 만큼 좁지 않은가
- **인쇄 일치도**: 화면 모드와 인쇄(PDF) 모드의 줄바꿈이 동일한가 — 다르면 둘 다 점검 필요

---

## 4. 검증자 보고 포맷

`line-break-verifier` 서브에이전트는 아래 형식으로 결과를 반환한다.

```
## Verdict: PASS | FAIL

### Hard Rules
- [PASS/FAIL] word-break: keep-all 적용 — (CSS 파일 경로:행 또는 미발견 사유)
- [PASS/FAIL] overflow-wrap 폴백 — (근거)
- [PASS/FAIL] line-break: strict — (근거)

### Quantitative (n/5 passed)
1. 어절 잘림: PASS/FAIL — (해당 슬라이드 번호와 잘린 어절. 예: "슬라이드 18 — '달라진|다.'")
2. 영문 단어 잘림: PASS/FAIL — (...)
3. 한 글자 미아: PASS/FAIL — (...)
4. <br/> 자연스러움: PASS/FAIL — (...)
5. 구두점 첫머리: PASS/FAIL — (...)

### Qualitative
- 의미 단위 일치: ...
- 컨테이너 폭: ...
- 인쇄 일치도: ...

### Required Fixes (FAIL인 경우만)
1. 슬라이드 X — 본문 컨테이너 폭 720→820px 로 확장 (어절 "달라진다" 보존)
2. 슬라이드 Y — `<br/>` 위치를 `"전략과 <br/>실행"` 사이로 이동
3. (전역) `colors_and_type.css` `html,body` 에 `word-break: keep-all` 누락 → 추가
```

---

## 5. 사용 흐름

1. 메인 에이전트가 `visual-verifier` 시각 검증을 먼저 통과시킨다.
2. 메인 에이전트가 슬라이드 PNG 캡처를 준비한다(이미 시각 검증 때 만든 것 재사용 가능).
3. `line-break-verifier` 서브에이전트 호출 — 캡처 + HTML 경로 + 본 루브릭 전달.
4. Verdict가 **PASS** 면 사용자에게 인계.
5. **FAIL** 이면 Required Fixes만 반영해 수정 → 캡처 재생성 → 다시 3번으로 (최대 3회 반복; 그 이상은 사용자 판단).

---

## 6. 일반적인 픽스 레시피

| 증상 | 권장 조치 |
| --- | --- |
| 컨테이너가 좁아 어절이 잘림 | `width`/`max-width` 를 어절 단위가 들어갈 만큼 확장 |
| 의미와 맞지 않는 위치에서 줄바꿈 | 의미 휴지에 명시적 `<br/>` 삽입 |
| 영문 식별자가 잘림 | 해당 영문 토큰을 `<span style="white-space:nowrap">` 로 묶기 |
| 구두점이 라인 첫머리로 밀림 | `line-break: strict` 확인, 또는 구두점 직전에 `&nbsp;` 삽입 |
| 인쇄에서만 잘림 | `@media print` 에서 컨테이너 폭/폰트 크기 재확인 |
