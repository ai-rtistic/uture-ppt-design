# Visual Rubric — UTURE 교안 PPT 검증 기준

이 문서는 산출물(HTML 슬라이드)이 **시각적으로 충분히 창의적이고 다양한지** 판정하기 위한 루브릭이다.

- 적용 대상: `./results/` 하위 모든 HTML 산출물
- 사용 주체: `visual-verifier` 서브에이전트 + 메인 에이전트의 자체 점검
- 우선순위: **하드 룰 위반은 무조건 Fail**. 그 외 항목은 점수 합산으로 판정.

---

## 0. 하드 룰 (위반 시 즉시 Fail)

[docs/system-prompt.md](system-prompt.md)의 UTURE 프로젝트 컨벤션을 그대로 적용한다.

- [ ] 표지·섹션·본문 어디에도 강의 분량(예: "30분", "60분 분량", "총 N분") 표기 없음
- [ ] 강사 표기가 필요한 경우 정확히 `uture | 유쳐 교육팀` 으로 표기 (다른 변형 금지)
- [ ] 텍스트 워드마크(`uture-text-logo`)는 표지 또는 클로징 중 한 군데, 또는 워크북 마지막 페이지에만. 헤더·섹션 디바이더 반복 노출 없음
- [ ] 인쇄용 페이지 규격(@page, A4/Letter, page-break) 적용됨

---

## 1. 정량 측정 항목 (각 항목 Pass/Fail)

| # | 항목 | 측정 방법 | 임계값 |
|---|------|-----------|--------|
| 1 | **레이아웃 다양성** | 전체 슬라이드에서 서로 구분되는 레이아웃 패턴 수 | 슬라이드 8장 이상이면 **3종 이상** / 4~7장이면 **2종 이상** |
| 2 | **연속 반복 금지** | 같은 레이아웃이 연속으로 나타나는 최대 횟수 | **연속 2장 이하** (3장 이상 동일 레이아웃 연속 시 Fail) |
| 3 | **텍스트-only 슬라이드 비율** | 비텍스트 요소(아이콘/도형/이미지/차트/다이어그램)가 없는 슬라이드 비율 | **≤ 20%** |
| 4 | **비텍스트 요소 밀도** | 각 본문 슬라이드에 시각 요소가 최소 1개 이상 존재 | 본문 슬라이드의 **≥ 80%** 충족 |
| 5 | **타이포 위계** | 한 슬라이드 내 폰트 사이즈 단계 수 | **2단 이상** (제목/본문 최소 구분) |
| 6 | **컬러 활용** | 디자인 시스템 팔레트에서 사용한 액센트 컬러 수(전체 덱 기준) | **2색 이상** (단색 톤다운 덱은 예외 명시 가능) |
| 7 | **구도 변화** | 좌-우 정렬, 중앙, 비대칭, 그리드 분할 등 구성 방식의 변화 | 전체 덱에서 **2가지 이상의 구도** 사용 |

> **합산 기준**: 7개 항목 중 **6개 이상 Pass** = 전체 Pass. 5개 이하면 Fail.

---

## 2. 정성 평가 항목 (각 항목 1줄 코멘트)

검증자는 아래 항목에 대해 한 줄씩 코멘트를 남긴다. 점수화하지 않지만, 정량 평가가 통과해도 **2개 이상에서 명백한 문제**가 있으면 Fail로 본다.

- **의도성**: 각 슬라이드의 시각 요소가 내용을 보조하는가, 단순 장식인가?
- **일관성**: 디자인 시스템(`./uture-design-system`)과 충돌하지 않는가?
- **여백**: 답답하거나 비어 보이지 않게 의도된 여백 운영인가?
- **AI 클리셰 회피**: 그라데이션 남용, 무의미한 아이콘 나열, 박스 안의 박스 패턴 등 흔한 AI 디자인 함정이 없는가?
- **인쇄 적합성**: PDF로 내보냈을 때 잘리거나 깨지는 요소가 없는가?

---

## 3. 검증자 보고 포맷

`visual-verifier` 서브에이전트는 아래 형식으로 결과를 반환한다.

```
## Verdict: PASS | FAIL

### Hard Rules
- [PASS/FAIL] 분량 표기 없음 — (근거)
- [PASS/FAIL] 강사 표기 — (근거)
- [PASS/FAIL] 워드마크 사용 — (근거)
- [PASS/FAIL] 인쇄 규격 — (근거)

### Quantitative (n/7 passed)
1. 레이아웃 다양성: PASS/FAIL — (관측값)
2. 연속 반복: PASS/FAIL — (관측값)
... (이하 동일)

### Qualitative
- 의도성: ...
- 일관성: ...
- 여백: ...
- AI 클리셰: ...
- 인쇄 적합성: ...

### Required Fixes (FAIL인 경우만)
1. 슬라이드 X — (구체 지시: "본문 3장 연속 좌측 정렬 → 가운데 1장은 비대칭 2-컬럼으로 변경")
2. ...
```

---

## 4. 사용 흐름

1. 메인 에이전트가 산출물을 `./results/`에 작성
2. 메인 에이전트가 슬라이드를 PNG로 캡처 (webapp-testing 스킬 또는 헤드리스 브라우저)
3. `visual-verifier` 서브에이전트 호출 — 캡처 + HTML 경로 + 이 루브릭 전달
4. Verdict가 **PASS**면 사용자에게 인계
5. **FAIL**이면 Required Fixes만 보고 수정 → 다시 2로 (최대 3회 반복; 그 이상은 사용자에게 판단 요청)
